# <a name="partners-user-guide-for-microsoft-365-app-compliance-program---saas"></a>Руководство пользователя партнера по программе соответствия Microsoft 365 приложений — SaaS

## <a name="1-overview"></a>1. Обзор 

Этот документ действует как пошаговое руководство для наших партнеров, зарегистрированных в рамках программы соответствия требованиям Microsoft 365, чтобы пройти Publisher аттестацию и сертификацию для своих приложений SaaS, используя портал Центра партнеров.

>[!IMPORTANT]
>Это руководство применимо только к приложениям SaaS, которые были утверждены и добавлены в список разрешений Microsoft 365 соответствия требованиям приложений. Если приложение не добавлено в список разрешений, вы не сможете продолжить работу в Центре партнеров. Чтобы включить SaaS в список разрешений, обратитесь appcert@microsoft.com.

## <a name="5---microsoft-365-renewal-workflow"></a>5. Microsoft 365 продления:
  
**Microsoft 365Publisher процесса аттестации и продления сертификации:**  

Microsoft 365 приложения теперь предлагает ежегодный процесс продления. В ходе этого процесса разработчики приложений могут обновить существующие анкеты Publisher аттестации и документы, необходимые для Microsoft 365 сертификации. 
 
**Преимущества:** 

- Сохраните эмблему сертификации в AppSource, Office Store, Teams Store и различных порталах администрирования, чтобы отличать ваше приложение от других. 
- Повышение уверенности клиентов в использовании сертифицированного приложения. 
- Помощь ИТ-администраторам в принятии обоснованных решений с помощью обновленных сведений о сертификации.

Новый процесс продления доступен в Центре [партнеров,](https://partner.microsoft.com/dashboard/home) чтобы обеспечить бесперебойную работу. Напоминание о продлении будет отображаться в Центре партнеров за 90 дней до даты окончания срока действия. Периодические напоминания также будут отправляться по электронной почте за 90, 60 и 30 дней до истечения срока действия. 
 
**Этап 1. Publisher аттестации:**
  
Ответы на Publisher аттестации приложения необходимо будет повторно отправить на ежегодной основе. Когда аттестация близко к отметке 1 года, по электронной почте будет отправлено напоминание о повторной отправке аттестации. 
 
**Шаг 1**. Выберите **"Продлить"**, чтобы продлить Publisher аттестации.
  
![Продление утверждено](../media/UserGuidePhotos/31.png)
  
**Шаг 2**. Просмотрите предыдущие Publisher аттестации и при необходимости обновите последние сведения. 
  
Отправьте Publisher аттестации для продления, когда все будет готово. Он будет проверяться аналитиком соответствия приложений M365.

![Обновление до аттестации](../media/UserGuidePhotos/29.png)
  
**Publisher утверждено продление аттестации:**
  
![Отправить на продление](../media/UserGuidePhotos/30.png)
  
**Publisher истек срок действия аттестации:**
  
Сведения о приложении необходимо обновить до даты окончания срока действия, чтобы сохранить страницу Publisher аттестации приложения в документации Майкрософт. Кроме того, во время обновления в различных онлайн-магазинах будут сохраняться недопустимые и значки для приложения. 
 
![Продление утверждено](../media/UserGuidePhotos/31.png)

**Примечание**. По истечении Publisher процесс продления аттестации можно запустить в любое время, нажав кнопку "Продлить".
 
**Этап 2. Microsoft 365 сертификации**
  
Сведения о сертификации приложения необходимо повторно отправить на ежегодной основе. Для этого потребуется повторная проверка элементов управления в области текущей среды. Когда сертификация скоро помечет 1 год, будет отправлено уведомление по электронной почте с подтверждением повторной отправки документов и свидетельства.
 
![Обновление аттестации](../media/UserGuidePhotos/32.png) 

**Сценарии утверждения или отклонения продления сертификации:**

**Сценарий 1.** 

Обновление сертификации запущено и находится на проверке.
 
![Обновление сертификата](../media/UserGuidePhotos/33.png) 

Сценарий 1A: 

Отклонение продления сертификации: 
- Сертификация может быть отклонена, если: 
     - Приложение не имеет необходимых инструментов, процессов или конфигураций и не сможет реализовать необходимые изменения в окне сертификации. 
     - Приложение имеет незавершенные уязвимости и не может быть исправлено в окне сертификации. 
 
![Отклонение сертификации](../media/UserGuidePhotos/34.png)

Сценарий 1B: 

Продление сертификации утверждено

![Утверждение продления сертификации](../media/UserGuidePhotos/35.png)

**Срок действия сертификации:**

Сведения о приложении необходимо обновить до даты окончания срока действия, чтобы сохранить страницу сертификации приложения в документации Майкрософт. Кроме того, во время обновления в AppSource и Team Store будут сохраняться некорректные действия и значки для приложения.

![Утверждение продления сертификации](../media/UserGuidePhotos/36.png)
  
Примечание. После истечения срока действия Publisher аттестацию и сертификацию можно запустить в любое время, нажав кнопку "Продлить".
