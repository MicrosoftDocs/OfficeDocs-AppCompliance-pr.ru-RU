Идентификатор команды отображается в наших журналах, когда с нашим приложением взаимодействуют в команде.
У нас есть ограниченный доступ к нашим производственным журналам для наших трех основателей, которые все базируются в США.

#### <a name="organizational-controls-for-data-stored-by-partner"></a>Организационный контроль за данными, хранящимися партнером

Опишите, как администраторы организации могут контролировать свою информацию в партнерских системах? например, удаление, удержание, аудит, архивирование, политика конечных пользователей и т.д.

>Приложение Cloverpop построено с использованием Ruby on Rails и размещается на Heroku PaaS (платформа как сервис), которая использует AWS для своей облачной инфраструктуры. У Heroku и AWS есть отчеты SOC, к которые можно получить доступ. Наше приложение использует PostgreS'L для шифрования в хранилище данных отдыха и является мульти-арендатором среды.
 
Весь наш код имеет автоматизированные тесты, написанные для него, который охватывает безопасность доступа к данным. Каждая сборка проходит строгий процесс проверки кода для обеспечения безопасности и ручной процесс тестирования, который также включает проверку подлинности пользователей и доступа к данным с помощью доступных действий пользователя. Существует четкое разделение между нашей производственной средой и всеми другими средами, такими как разработка и тестирование.
 
Только отдельный персонал имеет доступ к производственной среде и базе данных: учредители компании и небольшая горстка проверенных сотрудников, которые прошли проверку и имеют количественные потребности (такие как поддержка клиентов).
